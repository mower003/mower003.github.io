Milestone: CSUSM Course Scheduler Application

CS 441: Software Engineering Project

Github Link

Project Description

The goal of the “Milestone” application is to provide a personal planner for students of CSUSM. It is intended to allow users to store courses and tasks so that they have an idea of upcoming events. This application also allows users to subscribe to each others courses and consequently their tasks. These are all presented to users in an easy and intuitive way.

Technologies

  • AWS Amplify CLI
  • Dynamo DB (NoSQL)
  • Java
  • Android Studio
  • Amazon Cognito User-Pools

Architectural Model

The “Milestone” application used the Model-View-Controller-esque design pattern. This suits our application well because of the need for separate entities controlling data, presentation, and access to the server. The view will house the presentation, the controller will reside on the client and keep model up-to-date, and changes to the model will be propagated to the view.

Model

The implementation of the database was done using AWS DynamoDB using GraphQL which is NoSQL. This represents the model and had schemas for course, task,and userdata. Within each of these were the necessary variables related to each data point that was required for the implementation to work. Because of AWS Amplify connecting to the database was simply done through callback methods under a standard structure provided to us by the framework. This simplified querying, mutating, deleting, and updating information to and from the client.

View

The implementation of the view component involved the creation of layout files in Android Studio. These are comprised of XML and Java and are the front-end aspect of the implementation. These house the various UI components that users interact with to alter the state of the model or send information to the controllers. They may also be updated by the model via GraphQL calls.

Controller

The implementation of the controllers are Java files. These are tasked with creating, querying, or updating the model or updating the view with new information. These also hold information that is related to the state of the application. These establish connections to the model through GraphQL Callback methods. Some of the controllers lay within a designated controller class, while others reside in Activities where they have access to various functions of the underlying Android OS logic.
